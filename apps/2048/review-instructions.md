# 2048 게임 - Review 단계 지침

이 문서는 Review 서브에이전트 전용 작업 지침이다. Build 단계는 완료되었고 `apps/2048/index.html`, `apps/2048/style.css`, `apps/2048/game.js`가 이미 구현되어 있다. 너는 Build를 수행한 에이전트와는 별개의 에이전트로서, 코드를 새로 작성한 사람 입장이 아니라 검증자(QA) 입장에서 확인한다.

## 범위

- 검증 대상: `apps/2048/index.html`, `apps/2048/style.css`, `apps/2048/game.js`.
- 문제를 발견하면 **그 세 파일 안에서만** 직접 수정해도 된다. 그 외 블로그 파일은 건드리지 않는다.
- 검증 결과를 `apps/2048/review.md`에 작성한다.
- 참고 계획 문서: `apps/2048/spec.md` (요구사항 원본), `apps/2048/build-instructions.md` (Build 지침).

## 검증해야 할 것

### 1. 브라우저 동작 확인 (Browser 도구 사용)
- `preview_start`로 `apps/2048/index.html`을 파일 URL 또는 정적 서버로 열어서 실제로 확인한다 (예: `file:///C:/Users/buzru/OneDrive/바탕 화면/my_blog/apps/2048/index.html` 또는 필요하면 간단한 정적 서버를 통해).
- 방향키(ArrowUp/Down/Left/Right)로 타일이 밀리고 합쳐지는지 확인한다 (키보드 이벤트는 computer 도구의 key 액션으로 시뮬레이션).
- 점수(SCORE)가 병합 시 올라가는지, 새로고침 후에도 BEST 점수가 `localStorage`에서 유지되는지 확인한다.
- "새 게임" 버튼 클릭 시 보드/점수가 리셋되고 BEST는 유지되는지 확인한다.
- 콘솔 에러(`read_console_messages`)가 없는지 확인한다.
- `resize_window`로 모바일 뷰포트(예: 375x812)로 바꿔서 레이아웃이 깨지지 않는지, 보드가 화면 안에 들어오는지 확인한다.
- 다크/라이트 테마 전환이 있다면(테마 토글 UI가 이 게임 페이지엔 없을 수 있음 — spec.md 참고) `localStorage`의 `theme` 값을 바꿔서 재로드했을 때 테마가 반영되는지 확인한다.

### 2. 코드 리뷰
- `game.js`의 4방향 이동/병합 로직이 실제로 4방향 모두 올바르게 동작하는지(특히 회전/역회전 방향이 꼬여서 위/아래나 좌/우가 바뀌어 동작하는 흔한 버그가 없는지) 브라우저에서 각 방향키를 직접 눌러보며 확인한다.
- 유효하지 않은 이동(보드에 변화 없음)에서 새 타일이 생성되지 않는지 확인한다.
- 게임오버 판정과 승리(2048) 판정이 spec.md대로 동작하는지 확인한다(승리 후 "계속하기"로 이어서 진행 가능한지).
- 접근성: `role="grid"`/`row`/`gridcell`, `aria-live` 영역, `:focus-visible` 스타일이 실제로 존재하는지 `read_page`로 구조를 확인한다.
- `prefers-reduced-motion` 대응이 CSS에 실제로 있는지 확인한다.
- 블로그 다른 파일(css/, js/, templates/ 등)을 참조하거나 의존하지 않는지(자체 완결) 확인한다.

## 문제 발견 시

- 사소한 버그(예: 방향 로직 오류, aria-label 누락, 반응형 깨짐 등)는 해당 파일에서 직접 수정한다.
- 수정 후에는 반드시 브라우저에서 재확인해서 실제로 고쳐졌는지 검증한다.
- 구조적으로 spec.md와 크게 어긋나는 문제(예: 계획 자체를 다시 짜야 하는 수준)가 있다면 수정하지 말고 review.md에 이슈로 남기고 보고에서 강조한다.

## review.md 작성 형식

다음 항목을 포함해서 작성한다:
- 검증 일시, 검증 방법(사용한 도구)
- 항목별 결과 (동작 확인/코드 리뷰 체크리스트, 통과/실패 표시)
- 발견한 문제와 수정 내역 (있다면)
- 최종 결론: 배포(임베드) 가능 여부

## 작업 후 보고

review.md 작성 후, 발견한 문제와 수정 여부, 최종 결론을 한국어로 300단어 이내 요약해서 보고해라.
