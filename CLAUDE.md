# CLAUDE.md

이 파일은 Claude Code가 이 저장소에서 작업할 때 참고하는 가이드입니다.

## 프로젝트 개요

마크다운(.md) 파일을 읽어서 정적 블로그 웹사이트로 변환하는 프로젝트입니다.
별도의 빌드 프레임워크(React, Next.js, Vue, Hugo, Jekyll 등) 없이 순수 HTML, CSS, JavaScript로만 구현합니다.

## 핵심 요구사항

- **프레임워크 미사용**: npm 빌드 도구나 프론트엔드 프레임워크에 의존하지 않는다. 브라우저에서 바로 열리거나 정적 서버로 서빙 가능한 형태를 유지한다.
- **마크다운 변환**: `.md` 파일을 읽어 HTML로 렌더링한다. 마크다운 파싱은 가벼운 자체 구현 또는 단일 파일 형태의 외부 파서 사용을 우선 고려한다.
- **디자인 원칙**: 깔끔하고 읽기 좋은 타이포그래피 중심 레이아웃. 본문 가독성(줄 간격, 폭 제한, 여백)을 최우선으로 한다.
- **다크 모드**: 라이트/다크 테마를 모두 지원한다. `prefers-color-scheme`을 기본값으로 반영하고, 사용자가 수동으로 전환할 수 있는 토글을 제공한다. 선택한 테마는 로컬에 저장해 재방문 시 유지한다.
- **반응형 디자인**: 모바일 화면에서도 레이아웃, 폰트 크기, 여백이 자연스럽게 보이도록 한다.

## 아키텍처 방향

- 정적 파일 구조를 기본으로 한다 (예: `posts/*.md`, `index.html`, `post.html`, `css/`, `js/`).
- 마크다운 → HTML 변환은 클라이언트 사이드(브라우저에서 fetch 후 파싱) 또는 간단한 빌드 스크립트(Node.js 등 별도 런타임 스크립트, 프레임워크 아님) 중 하나로 진행하며, 구체적인 방식은 프로젝트 초기 설계 시 사용자와 협의한다.
- CSS는 CSS 변수(custom properties)를 활용해 라이트/다크 테마 색상을 관리한다.
- 외부 CDN 의존을 최소화하고, 꼭 필요한 경우가 아니면 라이브러리 없이 바닐라 JS로 구현한다.

## 작업 시 유의사항

- 새 기능을 추가할 때 불필요한 추상화나 빌드 단계를 도입하지 않는다.
- 접근성(a11y)을 고려한다 — 시맨틱 HTML 태그, 적절한 명암비(다크/라이트 모두), 키보드 네비게이션.
- 이 문서는 프로젝트가 진행되면서 실제 디렉터리 구조와 구현 방식이 확정되면 함께 업데이트한다.
